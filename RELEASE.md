# Релизный цикл и инструкция по публикации

## Релизный цикл

1. Обновите номер версии в `pyproject.toml`
2. Обновите номер версии в `setup.py` (если используется)
3. Обновите `CHANGELOG.md` с информацией о новом релизе
4. Создайте и запушьте git тег с новой версией
5. Опубликуйте пакет на PyPI

## Подготовка к публикации

1. Убедитесь, что все тесты проходят:
   ```
   python -m pytest tests/
   ```

2. Проверьте форматирование кода:
   ```
   ruff format .
   ```

3. Проверьте код на ошибки:
   ```
   ruff check .
   ```

4. Проверьте типизацию:
   ```
   mypy langchain_reindexer/
   ```

## Публикация на PyPI

1. Установите необходимые инструменты:
   ```
   pip install build twine
   ```

2. Соберите дистрибутив:
   ```
   python -m build
   ```

3. Проверьте дистрибутив:
   ```
   twine check dist/*
   ```

4. Опубликуйте на TestPyPI (для тестирования):
   ```
   twine upload --repository testpypi dist/*
   ```

5. Проверьте установку с TestPyPI:
   ```
   pip install --index-url https://test.pypi.org/simple/ langchain-reindexer
   ```

6. Если все работает корректно, опубликуйте на PyPI:
   ```
   twine upload dist/*
   ```

## После публикации

1. Создайте релиз на GitHub с тегом версии
2. Обновите документацию, если необходимо
3. Уведомите сообщество о новом релизе
